## Варианты улучшений (по убыванию отдачи)

### 1. CLAUDE.md, раздел «Сеть» — рецепт против прокси/TLS-граблей 🔥
**Проблема:** одна корневая причина (VPN TUN с подменой TLS + API-блокировки РФ) переоткрывалась минимум в 6 сессиях под разными масками: `unable to verify the first certificate` ×20 в Kwork-сессии, npm ECONNREFUSED ×15 в самса-проекте, Groq 403 ×4 в voice-core, SSL EOF на Bybit, `node --use-system-ca`. Это самый большой пожиратель токенов и времени за весь период.
**Вариант:** добавить в `jarvis-bot/CLAUDE.md` (и/или `~/.claude/CLAUDE.md`) блок:
- локальный трафик идёт через MITM-прокси → TLS-ошибки локально ожидаемы, сеть тестировать на Railway;
- node запускать с `--use-system-ca`;
- groq-sdk игнорирует HTTPS_PROXY/httpAgent → сразу undici ProxyAgent, проверку ключа пропустить;
- биржи/API блокируют РФ → тот же прокси-рецепт.

### 2. Скрипт `switch-proxy.ps1` — смена прокси вне Claude Code
**Проблема:** пользователь менял VPN-клиента дважды (NekoBox → Happ → sing-box). Сессия смены порта умерла мгновенно: Claude Code сам сидел на мёртвом прокси (курица-и-яйцо).
**Вариант:** PowerShell-скрипт (порт аргументом): обновляет `.npmrc`, `setx HTTPS_PROXY/HTTP_PROXY`, проверяет netstat. Запускается руками вне Claude Code.

### 3. Railway: один раз починить auth + записать координаты
**Проблема:** `Unauthorized` в MCP, `No linked project` (13 упоминаний `railway link` в одной сессии), интерактивный CLI виснет с exit 255. Терялись итерации в каждой деплой-сессии.
**Вариант:** выполнить `railway login` + `railway link` в jarvis-bot; записать в CLAUDE.md projectId (lucid-courtesy) и домен `jarvis-bot-production-90c2.up.railway.app`, чтобы MCP-вызовы шли с явным project_id.

### 4. CLAUDE.md, раздел «Правила домена» — то, что пользователь повторял голосом
**Проблема:** правила повторялись в разных сессиях и каждое нарушение стоило прерывания и выброшенной работы:
- «Kwork не скрейпить, только Gmail IMAP» — Claude дважды лез в Puppeteer, дважды прерван;
- «не запускать index.js с polling локально — 409-конфликт с Railway по токену» — повторено дважды, случился 409-шторм в проде;
- «jarvis-bot-dev — другой бот, не трогать».
**Вариант:** 4-5 строк в `jarvis-bot/CLAUDE.md`. Плюс удалить/архивировать дубликаты репо (`Desktop\jarvis-bot`, `Desktop\JARVIS`) — параллельные сессии в трёх клонах пушили в один remote и породили 409-шторм и получасовое расследование «чужих коммитов». В CLAUDE.md: «рабочая копия только `C:\Users\User10\jarvis-bot`».

### 5. Навык `deploy-and-verify-railway`
**Проблема:** цикл «код → push → Railway → логи → smoke-test» гонялся вручную десятки раз (551 ответ ассистента в Kwork-сессии).
**Вариант:** скил: push → `railway logs` с фильтром `[kwork]`/error → отправить `/kwork_check` → проверить ответ. Зависит от п.3 (рабочий railway auth).

### 6. Карта проектов в `~/.claude/CLAUDE.md`
**Проблема:** «где лежат проекты» спрашивалось отдельной сессией; ADMIN_CHAT_ID искали по папкам.
**Вариант:** таблица проектов (jarvis-bot, fly-bot, home-bot, makhachkala-home, crypto-signals) с путями и назначением — один раз в глобальный CLAUDE.md.

### 7. Мелкие исправления (пакетом за один заход)
- `PYTHONIOENCODING=utf-8` глобально — кракозябры cp866 в python-логах;
- ffmpeg в PATH (нужен для voice-пайплайна, `not recognized`);
- закоммитить `.claude/skills/security-review` — висит untracked с 6 июля;
- VS Code: закрывать diff-вкладки Claude Code перед выходом (источник «файл не найден» и unresponsive extension host); зафиксировать в память;
- поиск по репо тормозит из-за node_modules (ripgrep-таймауты в 2 сессиях) — проверить .gitignore/игнор-паттерны.

### 8. Привычки постановки задач (не требуют изменений в файлах)
- Целые задачи вместо микроменеджмента: сессия с драйвером IDT — 6+ сообщений «выполни эту команду»; «диагностируй микрофон» дало бы втрое короче.
- Образец хорошего промпта — fly-bot-сессия: нумерованное ТЗ + границы («не деплой, не коммить») = 0 ошибок.
- На больших билдах превью/скриншоты после каждого этапа, не в конце — лимит сессии съел результат самса-проекта (5MB, 451 ответ).

### 9. Навык-кандидат `new-project-check` (низкий приоритет)
**Проблема:** «сначала проверь существующие папки» пользователь формулирует руками в каждом новом проекте (магазин, крипто-бот).
**Вариант:** короткий скил или строка в глобальном CLAUDE.md. Частично закрывается п.6.

### 10. voice-core: блокер аппаратный, не программный
**Факт из 3 сессий подряд (~1.5MB транскриптов):** софт чист, драйвер стоит, но капсюль микрофона не получает bias-питание — голос не улавливается физически. Любая следующая софтверная итерация — потеря времени.
**Вариант:** USB-микрофон (обходит аналоговый тракт целиком). Это покупка, не код.
